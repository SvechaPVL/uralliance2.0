# Content Directory

Эта директория содержит все редактируемые данные для сайта в формате JSON.

## Файлы

### `contacts.json`
Контактная информация компании (телефон, email, адрес, мессенджеры).

**Структура:**
```json
{
  "company": "Название компании",
  "phone": {
    "display": "+7 (343) 123-45-67",
    "link": "tel:+73431234567"
  },
  "email": {
    "display": "info@example.com",
    "link": "mailto:info@example.com"
  },
  "office": {
    "address": "Адрес офиса",
    "mapLink": "https://yandex.ru/maps/..."
  },
  "messengers": {
    "whatsapp": {
      "number": "+73431234567",
      "defaultMessage": "Текст по умолчанию"
    },
    "telegram": {
      "username": "username",
      "defaultMessage": "Текст по умолчанию"
    }
  }
}
```

**Используется в:** ContactCTA секция, Footer

---

### `price-preview.json`
Карточки цен для главной страницы (секция PricePreview).

**Структура:**
```json
{
  "id": "уникальный-id",
  "title": "Название",
  "description": "Краткое описание услуги",
  "price": 50000,
  "priceFrom": true,         // если true, добавляет "от" перед ценой
  "unit": "₽",
  "badge": "Текст бейджа",
  "practice": "legal" | "tech" | "both",  // определяет цвета
  "featured": true,          // опционально, для выделения карточки
  "features": [
    "Пункт 1",
    "Пункт 2"
  ]
}
```

**Примечание:** Показывается ровно 3 карточки в порядке из файла.

---

### `prices.json`
Полный прайс-лист для страницы `/price`.

**Структура:**
```json
{
  "id": "уникальный-id",
  "practice": "legal" | "tech",
  "category": "Категория",
  "title": "Название услуги",
  "description": "Подробное описание",
  "price": 50000,
  "priceFrom": true,
  "unit": "проект" | "месяц" | "консультация",
  "featured": true,
  "features": [
    "Что входит 1",
    "Что входит 2"
  ]
}
```

## Как редактировать

1. **Откройте нужный JSON файл** в редакторе
2. **Отредактируйте значения** (цены, тексты, описания)
3. **Сохраните файл** - изменения применятся автоматически после пересборки

## Правила

- ✅ Всегда используйте валидный JSON (проверяйте запятые!)
- ✅ Цены указывайте числом без пробелов: `50000`, не `"50 000"`
- ✅ Для `practice` используйте только: `"legal"`, `"tech"`, или `"both"`
- ✅ ID должны быть уникальными
- ❌ Не удаляйте обязательные поля
- ❌ Не меняйте структуру объектов

## Цветовая логика

- **`practice: "legal"`** → золотые цвета (юридические услуги)
- **`practice: "tech"`** → голубые цвета (IT-услуги)
- **`practice: "both"`** → градиент legal + tech (интеграция)
- **`featured: true`** → выделяет карточку (обычно tech вариант)
